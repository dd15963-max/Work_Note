# Work Note

로컬 PC에서 바로 실행하는 개인용 업무 메모장입니다. 서버, 로그인, 데이터베이스, 빌드 과정 없이 `index.html` 파일을 브라우저에서 열어 사용합니다.

## 실행 방법

1. `sales-note-app` 폴더를 엽니다.
2. `index.html` 파일을 더블클릭합니다.
3. 브라우저에서 메모를 입력하고 저장합니다.

인터넷 연결이 없어도 기본 기능은 동작합니다.

## 기본 사용 흐름

1. 처음 열면 `일정` 포탈이 기본으로 표시됩니다.
2. 상단 탭에서 `일정`, `영업`, `정산`, `출력`, `기타` 포탈을 전환합니다.
3. `영업`에서는 기존 고객사별 영업 메모와 첨부자료를 관리합니다.
4. `정산`에서는 분할 결제와 선금 결제처럼 계속 챙겨야 하는 입금/차감 업무를 관리합니다.
5. `일정`에서는 영업, 정산, 출력, 기타 업무의 날짜가 있는 항목을 월간 캘린더와 오늘 일정으로 확인합니다.
6. `출력`에서는 관련 업체, 담당자(요청자), 기한 범위, 출력 종류를 관리하고, 각 업무의 `파일함`에서 출력 파일과 파일별 메모를 관리합니다.
7. `기타`에서는 영업, 정산, 출력에 속하지 않는 일반 업무를 기록하고, 각 업무의 `파일함`에서 관련 파일을 관리합니다.

## 파일 구조

```text
sales-note-app/
├─ index.html
├─ style.css
├─ script.js
└─ README.md
```

- `index.html`: 화면 구조
- `style.css`: 대시보드, 입력 폼, 목록 화면 스타일
- `script.js`: 메모 저장, 검색, 필터, 백업, 불러오기 로직
- `README.md`: 실행 및 관리 안내

## 주요 기능

- 고객사별 영업 메모 등록, 수정, 삭제
- 기본 화면은 목록 중심으로 넓게 사용하고, `새 메모`, `새 출력 업무`, `새 기타 업무` 또는 목록의 `수정` 클릭 시 상세/수정 패널 열기
- 상단 포탈 탭: `일정`, `영업`, `정산`, `출력`, `기타`
- 통합 일정 캘린더에서 `[영업] 고객사 연락`, `[영업] 고객사 미팅`, `[정산] 업체 처리`, 출력 기한, 기타 업무 기한 확인
- 통합 일정 옆 `오늘 일정` 패널에서 오늘 할 일을 자세히 확인
- 공통 파일함: 영업, 정산, 출력, 기타 업무 건별로 오른쪽 파일함을 열어 여러 파일 업로드, 파일별 메모 입력, 다운로드, 삭제, 파일 정보 수정
- 정산 업무: 관련 업체, 담당자, 관련 영업 메모, 분할 결제/선금 결제 유형, 회차별 결제 일정표 또는 선금 차감 목록, 잔액, 다음 처리일, 파일함 관리
- 출력 업무: 관련 업체, 담당자(요청자), 기한 범위, 진행 상태, 출력 종류, 파일함 기반 출력 파일 관리
- 기타 업무: 업무명, 구분, 담당/요청자, 기한 범위, 진행 상태, 파일함 기반 관련 파일 관리
- 고객사명, 담당자명, 연락처, 이메일, 관심 장비/소재, 진행 상태, 중요도, 미팅 일자, 다음 액션, 연락 예정일, 메모 관리
- 진행 상태와 중요도 필터
- 고객사, 담당자, 연락처, 이메일, 메모 통합 검색
- 중요도순, 다음 연락일, 최종 수정일, 고객사명 기준 정렬과 별도 화살표 방향 전환
- 목록에서 관심제품, 견적여부, 구매가능성, 관련자료, 다음연락, 미팅일정, 최근연락을 바로 확인
- 상단 요약 카드 클릭으로 전체 메모, 이번 주 연락/미팅 예정, 미팅 예정, 기한 지남 항목만 빠르게 보기
- 전체 메모, 기한 지난 팔로업, 이번 주 예정, 미팅 예정 요약
- 기한 지난 팔로업, 오늘 연락, 이번 주 연락, 미팅 예정 구역별 보기
- `미팅 예정` 상태 선택 시 미팅 일자 입력
- 다음 연락 예정일을 날짜 또는 `미정`으로 관리
- 예상 매출, 매출 금액, 매출 구분 관리
- 업체 메모별 첨부자료 업로드, 목록 확인, 다운로드, 삭제
- 이미지 캡처 파일은 첨부자료 목록에서 작은 미리보기 표시
- 브라우저 `localStorage` 자동 저장
- 작성 중인 입력 내용 초안 자동 저장
- 저장하지 않은 작성 내용이 있을 때 브라우저 종료 경고
- `백업 관리` 메뉴에서 기록만 백업, 전체 백업, 교체 불러오기, 병합 불러오기
- 마지막 백업 날짜 표시와 백업 리마인더
- CSV 내보내기
- 전체 메모장 초기화
- 모바일 화면에서 카드형 메모 목록, 상단 고정 탭, 고정 새 업무 버튼 표시

## 데이터 백업 방법

### 기록 백업 JSON

1. 화면 상단의 `백업 관리` 메뉴를 열고 `기록만 백업` 버튼을 클릭합니다.
2. `sales-note-backup-YYYYMMDD-HHMM.json` 형식의 파일이 다운로드됩니다.
3. 백업 파일은 회사에서 허용된 OneDrive, NAS, Drive 등 안전한 위치에 직접 보관합니다.

기록 백업 JSON에는 앱 버전, 백업 생성일, 상태 옵션, 영업/정산/출력/기타 업무 데이터와 첨부자료 목록 정보가 포함됩니다.

첨부파일 원본은 JSON 백업에 포함되지 않고 브라우저 IndexedDB에 저장됩니다. 다른 PC나 다른 브라우저로 옮길 때는 파일 원본을 별도로 보관해야 합니다.

### 전체 백업 ZIP

1. 화면 상단의 `백업 관리` 메뉴를 열고 `전체 백업` 버튼을 클릭합니다.
2. `work-note-full-backup-YYYYMMDD-HHMM.zip` 형식의 파일이 다운로드됩니다.
3. ZIP 안에는 `backup.json`과 `attachments/` 폴더가 함께 들어갑니다.
4. `attachments/` 폴더에는 영업, 정산, 출력, 기타 업무에 연결된 원본 파일이 업무별 폴더로 저장됩니다.

전체 백업 ZIP은 원본 파일 장기 보관과 PC 이관용입니다. ZIP 안의 `backup.json`에는 각 첨부자료의 `backupPath`가 기록되어 원본 파일 위치를 확인할 수 있습니다.

### 전체 백업 ZIP 불러오기

1. 화면 상단의 `백업 관리` 메뉴를 열고 `전체 교체` 버튼을 클릭합니다.
2. 이전에 저장한 `work-note-full-backup-YYYYMMDD-HHMM.zip` 파일을 선택합니다.
3. 기존 업무 데이터와 첨부파일 원본 저장소를 덮어쓸지 확인합니다.
4. 불러오기가 완료되면 ZIP 안의 `backup.json`은 업무 데이터로 복원되고, `attachments/` 원본 파일들은 브라우저 IndexedDB에 다시 저장됩니다.

다른 PC에서 사용할 때는 원본 파일 폴더를 직접 특정 위치에 복사할 필요 없이 전체 백업 ZIP 파일 하나를 선택하면 됩니다.

### 전체 백업 ZIP 병합

1. 화면 상단의 `백업 관리` 메뉴를 열고 `전체 병합` 버튼을 클릭합니다.
2. 이전에 저장한 `work-note-full-backup-YYYYMMDD-HHMM.zip` 파일을 선택합니다.
3. 같은 ID의 업무가 이미 있으면 기존 건 안에서 내용을 스마트 병합합니다.
4. 메모는 중복 줄이나 중복 구분 항목을 하나만 남기고 합산합니다.
5. 다음 액션, 상태, 날짜, 담당자처럼 하나의 현재값만 가져야 하는 필드는 가져오는 백업 값을 우선 적용합니다.
6. 새 업무와 새 첨부파일은 현재 데이터에 추가합니다.

ZIP 병합은 PC와 핸드폰, 다른 브라우저에서 따로 쌓인 데이터를 회사 PC 데이터에 붙일 때 사용합니다. 첨부파일 ID나 파일명/용량이 같은 파일은 중복 저장하지 않고, 같은 첨부파일의 메모/자료 종류/발송일은 가져오는 백업 값을 우선 적용합니다.

## JSON 불러오기 방법

1. 화면 상단의 `백업 관리` 메뉴를 열고 `기록만 교체` 버튼을 클릭합니다.
2. 이전에 저장한 JSON 백업 파일을 선택합니다.
3. 기존 데이터를 덮어쓸지 확인하는 창에서 승인합니다.
4. 불러오기가 완료되면 목록이 새 데이터로 갱신됩니다.

불러오기 직전의 현재 데이터는 브라우저 localStorage의 임시 백업 키에 한 번 저장됩니다.
또한 기존 데이터를 덮어쓰기 전에 `sales-note-pre-import-backup-YYYYMMDD-HHMM.json` 파일을 자동으로 다운로드합니다.

## JSON 병합 방법

1. 화면 상단의 `백업 관리` 메뉴를 열고 `기록만 병합` 버튼을 클릭합니다.
2. 이전에 저장한 JSON 백업 파일을 선택합니다.
3. 같은 ID의 업무가 이미 있으면 기존 건 안에서 내용을 스마트 병합합니다.
4. 새 업무는 현재 데이터에 추가합니다.

JSON 백업에는 원본 파일이 포함되지 않으므로 병합된 첨부자료는 파일명, 메모, 날짜 같은 기록만 추가됩니다. 원본 파일 다운로드까지 유지하려면 `전체 백업 ZIP`과 `ZIP 병합`을 사용합니다.

## 작성 중 초안

입력 폼에 작성 중인 내용이 있으면 브라우저 localStorage에 초안이 자동 저장됩니다. 저장하지 않은 상태로 브라우저를 닫으려 하면 경고가 표시됩니다.

앱을 다시 열었을 때 저장되지 않은 초안이 있으면 복원 여부를 묻습니다.

## CSV 내보내기

`백업 관리` 메뉴의 `CSV` 버튼을 누르면 엑셀에서 열 수 있는 CSV 파일이 다운로드됩니다. 한글이 깨지지 않도록 UTF-8 BOM을 포함합니다.

CSV에는 첨부자료 개수와 첨부 파일명 목록도 포함됩니다.

## 정산 업무 관리

`정산` 탭은 분할 결제와 선금 결제처럼 일정 기간 동안 계속 확인해야 하는 금액 업무를 따로 관리합니다.

- `분할 결제`는 `총 관리 금액 - 회차 입금 완료액 - 기타 차감 금액` 기준으로 잔여 금액을 계산합니다.
- `선금 결제`는 `선금 금액 - 차감 완료액` 기준으로 선금 잔액을 계산합니다.
- 회차별 결제 일정 또는 선금 차감 목록이 있으면 캘린더와 오늘 일정에 `[정산] 업체명 회차/차감`으로 표시됩니다.
- 시작일, 간격, 회차/차감 수, 금액을 넣어 일정을 자동 생성하거나, 엑셀에서 복사한 표를 붙여넣어 불러올 수 있습니다.
- 시작일을 비우고 `현재 회차/총 회차`와 `다음 처리일`을 입력하면, 다음 처리일을 다음 회차 날짜로 보고 시작일을 역산해 전체 일정을 채웁니다.
- 분할 결제에서는 회차 상태를 `결제 완료`로 바꾸면 회차 입금 완료액이 반영되고, 선금 결제에서는 차감 상태를 `차감 완료`로 바꾸면 차감 완료액이 반영됩니다.
- 계약서, 발주서, 세금계산서, 입금증, 메일 캡처 등은 각 정산 업무의 `파일함`에서 관리합니다.

## 전체 메모장 초기화

`백업 관리` 메뉴의 `전체 메모장 초기화` 버튼을 누르면 확인창이 표시됩니다. 확인하면 현재 브라우저에 저장된 영업, 정산, 출력, 기타 업무와 관련 파일 원본이 모두 삭제되고 새 메모장 상태로 돌아갑니다.

초기화 전에 필요한 자료는 `전체 백업`으로 먼저 보관하는 것을 권장합니다.

## 첨부자료 관리

1. 메모를 먼저 저장하거나 기존 메모를 엽니다.
2. 우측 상세/수정 패널의 `첨부자료` 영역에서 파일을 선택합니다.
3. 자료 종류, 발송일, 파일 메모를 입력합니다.
4. `첨부자료 저장` 버튼을 누릅니다.
5. 저장된 첨부자료는 목록에서 `다운로드` 또는 `삭제`할 수 있습니다.

PDF, PPT, Word, Excel, 이미지 캡처, ZIP 등 파일 종류를 제한하지 않습니다. 이미지 파일은 목록에서 작은 미리보기를 표시합니다.

## 주의사항

- 실제 영업 데이터 파일을 프로젝트 폴더에 고정 저장하지 않습니다.
- GitHub 저장소에는 코드만 저장하고 실제 고객 데이터는 올리지 않습니다.
- 브라우저의 사이트 데이터 또는 캐시를 삭제하면 localStorage 데이터가 사라질 수 있습니다.
- 브라우저 사이트 데이터를 삭제하면 IndexedDB에 저장된 첨부파일 원본도 사라질 수 있습니다.
- 중요한 데이터는 주기적으로 `기록만 백업` 또는 `전체 백업`으로 별도 위치에 보관해야 합니다.
- 기록 백업 JSON은 첨부파일 원본을 포함하지 않습니다. 원본 파일까지 장기 보관하려면 `전체 백업 ZIP`을 사용합니다.
- 다른 PC로 옮길 때 원본 파일까지 복원하려면 `전체 교체` 또는 `전체 병합`을 사용합니다.
- 백업을 7일 이상 하지 않으면 화면 상단에 백업 리마인더가 표시됩니다.
